# `report/`

Working folder for the EEEM068 group report.

We are writing the report itself in Overleaf, not here. This folder is just so we have a shared plan and a single source of truth for the bibliography.

| File | Purpose |
|---|---|
| `OUTLINE.md` | Who leads each section, what each section needs to cover, the figure list, and the outstanding TODOs. Read this first. |
| `references.bib` | The bibliography. Add new entries here first, then mirror them into Overleaf. |

The Overleaf link goes in `OUTLINE.md` once Talha sets the project up.

## Reminders from the coursework brief

- The report is IEEE conference, two-column, **5 pages excluding references**. Appendices for figures and extra plots are unlimited and do not count.
- Every member contributes to literature review, methodology, coding, training, evaluation, and writing. Lead-author labels in `OUTLINE.md` are just a first-pass plan, not exclusive ownership.
- Anything we got help with from an LLM has to be read line-by-line and fully understood; the oral exam can ask any of us to derive or modify any part of the report or code.
- Creativity is 20% of the report mark and is graded across the whole thing. Tag creative bits in the LaTeX source with `% creativity: short label` so we can spot them at the end.
